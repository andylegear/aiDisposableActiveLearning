AI-Generated Disposable Software as Active Learning Instruments in CS Education

Andrew Le Gear · ABCD Research Group, Lero · University of Limerick · 2026

This study investigates whether large language models can generate disposable educational software — small, single-purpose, browser-based tools that teach one concept and are designed to be discarded rather than maintained. Five artifacts were produced across five CS modules in approximately 3.3 hours of wall-clock time, with zero lines of code written by hand. Each artifact is evaluated using a novel composite metric, the Disposable Software Quality Index (DSQI), combining automated code analysis with structured reviews from expert practitioners and module coordinators.

Reproducibility

This repository is a self-contained data pack designed so that any researcher can independently verify or extend the findings. It includes: the complete source code for all five artifacts exactly as generated by the AI; timestamped development logs and WakaTime session data recording every prompt, failure, and edit; raw static-analysis output (cloc, cyclomatic complexity) used to compute the automated DSQI sub-scores; the evaluation instruments themselves (expert review and module coordinator surveys) deployed as client-side web tools that produce structured JSON; and the Python scripts used to collect and compute the metrics. No external databases or API keys are required — clone the repository and every input to the DSQI formula is present and re-computable.
